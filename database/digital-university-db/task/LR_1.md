## **🔬 Лабораторна робота №1**

### **Аналіз предметної області та ER-діаграма**

### **Мета**

Проаналізувати предметну область «Цифровий університет» та побудувати ER-діаграму.

### **Завдання**

1.  Виділити сутності:

    - Faculty

    - Department

    - Teacher

    - Student

    - Course

    - Enrollment

    - Group

2.  Для кожної сутності визначити:

    - атрибути;

    - первинний ключ.

3.  Визначити зв’язки між сутностями та їх кратність.

4.  Побудувати ER-діаграму.

5.  Відобразити, що один зі студентів може бути старостою групи.

### **Результат**

ER-діаграма у вигляді файлу або зображення.

| Зв’язок | Кратність | Пояснення |
|---|---:|---|
| faculty → department | 1:M | Один факультет може містити багато кафедр. |
| department → teacher | 1:M | Одна кафедра може мати багато викладачів. |
| department → student_group | 1:M | Одна кафедра може мати багато академічних груп. |
| student_group → student | 1:M | Одна академічна група може містити багато студентів. |
| department → course | 1:M | Одна кафедра може відповідати за багато дисциплін. |
| teacher → course | 1:M | Один викладач може викладати багато дисциплін. |
| student_group → course | 1:M | Одна академічна група може вивчати багато дисциплін. |
| student → enrollment | 1:M | Один студент може мати багато записів на дисципліни. |
| course → enrollment | 1:M | Одна дисципліна може мати багато записів студентів. |
| student ↔ course | M:M | Багато студентів можуть вивчати багато дисциплін; зв’язок реалізовано через Enrollment. |
| student_group → student / monitor_student_id | 1:0..1 | У групи може бути один староста, який є студентом цієї групи. |
