# University DB Practice

This folder contains six PostgreSQL practice assignments based on a simple academic database context. The goal is to train SQL mechanics through small end-to-end tasks: creating tables, inserting and changing data, filtering results, joining tables, grouping data, and writing subqueries.

## Learning Focus

The practices are intentionally simple and procedural. Each task asks you to read the requirement, choose the SQL operation, run the query, inspect the result, and adjust if needed.

The focus is repetition, correctness, and confidence with SQL syntax. This makes the folder useful as a training layer before larger database projects.

## PostgreSQL Conventions

The practices use PostgreSQL-oriented syntax and habits:

- `GENERATED BY DEFAULT AS IDENTITY` for generated keys;
- real `DATE` columns instead of date strings;
- `VARCHAR`, `NUMERIC`, and explicit constraints;
- `RETURNING` for checking changed rows;
- transactions with `BEGIN`, `COMMIT`, and `ROLLBACK`;
- `TRUNCATE TABLE ... RESTART IDENTITY`;
- `ILIKE`, `FILTER`, and `CASE` where they fit naturally.

## Suggested Workflow

1. Create a local PostgreSQL database.
2. Open DBeaver, pgAdmin, `psql`, or another database tool.
3. Go through the practices in order.
4. Keep each solution as a separate SQL script or clearly separated query block.
5. Add short comments before important query groups.
6. Run verification `SELECT` queries after changes.
7. Use transactions when testing risky `UPDATE`, `DELETE`, or `TRUNCATE` statements.
