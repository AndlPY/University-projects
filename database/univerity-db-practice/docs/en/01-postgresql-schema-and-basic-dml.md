- Practice PostgreSQL data types such as `BIGINT`, `VARCHAR`, `DATE`, and `NUMERIC`.

## PostgreSQL Notes

- PostgreSQL folds unquoted identifiers to lowercase, so this practice uses `snake_case` table and column names.
- Use `GENERATED BY DEFAULT AS IDENTITY` for an auto-generated numeric primary key.
- PostgreSQL has a real `DATE` type, so dates should be stored as dates, not as text.
- `NUMERIC(5,2)` is a good fit for scores when exact decimal values are needed.
- Foreign key checks are enabled by default in PostgreSQL.
- `ALTER TABLE` supports adding, dropping, renaming, and changing columns, but every structure change should still be intentional.
- `RETURNING` can be used after `INSERT`, `UPDATE`, or `DELETE` when you need to see affected rows immediately.

## Table Schema

Create the `students` table with the following columns and constraints.

| Column | Type and Constraint | Notes |
| --- | --- | --- |
| `student_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` | Unique student identifier |
| `first_name` | `VARCHAR(100) NOT NULL` | Student first name |
| `last_name` | `VARCHAR(100) NOT NULL` | Student last name |
| `birth_date` | `DATE` | Student birth date |
| `email` | `VARCHAR(150) UNIQUE` | Unique email address |
| `average_score` | `NUMERIC(5,2) CHECK (average_score >= 0 AND average_score <= 100)` | Average score from 0 to 100 |
| `status` | `VARCHAR(30) DEFAULT 'Active'` | Default student status |

## Tasks

### Part 1. Create the Table

1. Write the SQL query that creates the `students` table.
2. Make sure every constraint from the schema above is included in `CREATE TABLE`.
3. Use lowercase `snake_case` identifiers.

### Part 2. Insert Data

1. Add at least 5 students.
2. In two records, do not provide the `status` field so the `DEFAULT` value can be checked.
3. Add at least one student with `average_score > 90`.
4. Use realistic data: Ukrainian names, birth dates, and email addresses.
5. Use an explicit list of columns in each `INSERT`.

### Part 3. Manipulate Data

1. Display all students.
2. Display students with `average_score > 85`.
3. Display only `first_name` and `email`.
4. Update `average_score` for one student by `student_id`.
5. Update one student's `status` to `Inactive`.
6. Delete one student by `student_id`.
7. Use `RETURNING *` for at least one `UPDATE` or `DELETE`.

### Part 4. Modify the Table Structure

Add two new columns:

- `phone_number` - `VARCHAR(30)`
- `group_name` - `VARCHAR(50)`

Use the `ALTER TABLE ... ADD COLUMN` syntax.

### Part 5. Final Verification

1. Display the final table state:

```sql
SELECT *
FROM students;
```

2. Count active students:

```sql
SELECT COUNT(*)
FROM students
WHERE status = 'Active';
```
